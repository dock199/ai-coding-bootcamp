# 日報自動生成アプリ — 要件定義書

## 1. プロジェクト概要

PCの作業画面スクリーンショットをアップロードすると、Azure OpenAI GPT-4.1（マルチモーダル）を活用してOCR不要で日報を自動生成するWebアプリケーション。  
データベースを使用しないセッションベースのMVP（Minimum Viable Product）として構築する。

### 背景・目的

- 日報作成は毎日の定型業務であり、手入力に時間がかかる
- スクリーンショットから自動で日報を生成し、日報作成の時間を大幅に削減する
- マルチモーダルモデルを活用することで、OCRエンジンの導入・管理コストを回避する

### ユーザーストーリー

> **利用者**として、PCの作業画面スクリーンショットをアップロードすると、AIが自動で日報を生成してくれるので、日報作成の時間を大幅に削減できる。

## 2. 機能要件（EARS記法）

| ID | 種別 | 要件 |
|----|------|------|
| FR-01 | イベント駆動 | ユーザーが画像をドラッグ&ドロップまたはクリップボード貼り付けしたとき、システムは画像をアップロードエリアに表示すること |
| FR-02 | イベント駆動 | ユーザーが複数の画像をアップロードしたとき、システムはすべての画像を一覧表示し、個別に削除できること |
| FR-03 | イベント駆動 | ユーザーが「日報生成」ボタンを押したとき、システムはアップロードされた画像をバックエンドに送信し、日報を生成すること |
| FR-04 | 状態駆動 | 日報生成処理中の間、システムはローディングインジケーターを表示すること |
| FR-05 | イベント駆動 | 日報生成が完了したとき、システムはマークダウン形式で「作業内容」「進捗状況」「課題・問題点」の3セクション構成の日報を表示すること |
| FR-06 | イベント駆動 | ユーザーが「コピー」ボタンを押したとき、システムは日報テキストをクリップボードにコピーすること |
| FR-07 | イベント駆動 | ユーザーが日報を編集したとき、システムは編集内容をリアルタイムに反映すること |
| FR-08 | 望ましくない動作 | 画像未アップロードで「日報生成」が押された場合、システムは画像アップロードを促すメッセージを表示すること |
| FR-09 | 望ましくない動作 | APIエラーが発生した場合、システムはわかりやすいエラーメッセージを表示すること |
| FR-10 | イベント駆動 | バックエンドが画像を受信したとき、Azure OpenAI GPT-4.1 に画像を送信し日報テキストを生成すること |
| FR-11 | イベント駆動 | バックエンドがAPI応答を受信したとき、マークダウン形式の日報をフロントエンドに返すこと |

## 3. 非機能要件（EARS記法）

| ID | 種別 | 要件 |
|----|------|------|
| NF-01 | 普遍的 | システムはモダンブラウザ（Chrome, Firefox, Safari, Edge最新版）で動作すること |
| NF-02 | 普遍的 | システムは画像のリサイズ・圧縮を行い、APIリクエストサイズを最適化すること |
| NF-03 | 普遍的 | システムは環境変数（`.env`）でAzure OpenAI APIキー・エンドポイントを管理すること |
| NF-04 | 普遍的 | システムはCORS設定によりフロントエンドからのリクエストのみ受け付けること |
| NF-05 | 望ましくない動作 | アップロードされた画像が不正な形式の場合、システムは適切なエラーメッセージを返すこと |
| NF-06 | 普遍的 | システムはアップロードされた画像を永続的に保存せず、処理後に破棄すること |
| NF-07 | 普遍的 | `.env`ファイルは`.gitignore`に追加し、リポジトリに含めないこと。参考用に`.env.example`をリポジトリに含めること |
| NF-08 | 普遍的 | 本番環境（Azure App Service）ではアプリケーション設定で環境変数を管理すること |

## 4. スコープ外

以下はMVPスコープ外とし、将来的な拡張として検討する：

- モバイル対応（デスクトップブラウザのみ対象）
- IE11ブラウザ対応
- データベースによる永続化
- ユーザー認証・認可
- 日報テンプレートのカスタマイズ機能
- ファイルダウンロード機能（マークダウンファイルとしてのエクスポート）
- バックアップ案としてのOCR利用

## 5. 日報出力形式

### テンプレート構成

生成される日報は以下の3セクション構成（マークダウン形式）：

```markdown
## 作業内容
- （スクリーンショットから読み取った作業内容）

## 進捗状況
- （作業の進捗に関する記述）

## 課題・問題点
- （発見された課題や問題点）
```

### 出力・利用方法

- 画面上にマークダウンテキストとして表示
- テキストエリアで直接編集可能
- クリップボードへのワンクリックコピー

## 6. 制約事項

| 項目 | 制約 |
|------|------|
| 開発期間 | 2週間でMVP完成 |
| データベース | 使用しない（セッションベースの一時保存のみ） |
| 画像保存 | 永続保存しない（処理後に破棄） |
| 対応ブラウザ | モダンブラウザのみ（IE11非対応） |
| デプロイ先 | Azure App Service（FastAPI） |
| AIモデル | Azure OpenAI GPT-4.1（マルチモーダル） |
